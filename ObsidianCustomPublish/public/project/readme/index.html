Jeff’s Link - Personal Knowledge Base #

A personal knowledge base built with Hugo using the Book theme, deployed via AWS Amplify Console.

Version Information #

  • Hugo Version: 0.145.0
  • Theme: Hugo Book Theme (latest)
  • Node.js: 18.x or later
  • AWS Amplify: Latest

Introduction #

1.1 Overview #

This is a personal knowledge base that:

  • Uses Hugo as the static site generator
  • Implements the Hugo Book theme
  • Deploys via AWS Amplify Console
  • Supports both light and dark themes
  • Includes journal and download sections

1.2 Features #

  • Automated deployments via AWS Amplify
  • CloudFront CDN for fast content delivery
  • Image optimization
  • Responsive design
  • Dark mode support
  • Search functionality
  • Journal entries with date-based navigation
  • Download section for resources

1.3 Target Audience #

  • Personal knowledge management
  • Technical documentation
  • Journal/blog content
  • Resource sharing

Development Guide #

5.1 Getting Started # 

# Clone repository
git clone [your-repo-url]
cd [repo-name]
git submodule update --init --recursive

# Install dependencies
npm install

# Start development server
hugo server -D

5.2 Development Workflow #

  1. Create new content:

    # New journal entry
hugo new content journal/2024-03-07.md

# New download
cp file.pdf content/_downloads/

  2. Edit content:

    • Use your preferred editor
    • Preview changes with hugo server -D
    • Test locally before committing

  3. Build and test:

    # Build site
./scripts/build.sh

# Check for issues
hugo --check

5.3 Navigation and Templates #

The site uses custom templates for consistent navigation and link handling:

  • layouts/_default/single.html: Main template for single pages
  • layouts/partials/journal-nav.html: Handles journal entry navigation
  • layouts/partials/_markup/render-link.html: Processes internal links with case-insensitive handling

Key features:

  • Case-insensitive link handling for journal entries
  • Automatic previous/next navigation for journal entries
  • Consistent URL formatting across the site
  • Custom styling for journal links

5.4 Theme Customization #

  • Custom styles: assets/scss/_custom.scss
  • Theme settings: config/_default/config.toml
  • Custom templates: layouts/

Build Process #

Local Build # 

# Development build
hugo server -D

# Production build
hugo --minify --theme hugo-book

Build Features #

  • Asset Pipeline: Hugo’s built-in asset pipeline handles SCSS compilation and optimization
  • Image Optimization: Automatic image optimization for JPG and PNG files
  • Minification: HTML, CSS, and JavaScript minification
  • Cache Management: Efficient caching for faster rebuilds

Build Requirements #

  • Hugo Extended (for SCSS processing)
  • ImageMagick (optional, for image optimization)

Build Output #

  • Generated site in public/ directory
  • Optimized assets in public/assets/
  • Minified HTML, CSS, and JavaScript

Documentation Integration #

12.1 Automatic README Integration #

The project includes an automatic documentation integration feature that copies the main README.md file into the Hugo content structure during deployment. This ensures that:

  • Project documentation is always up to date
  • Documentation is accessible through the Hugo site navigation
  • Documentation is searchable using the site’s search functionality
  • Documentation is properly formatted with the Hugo Book theme

The integration process:

  1. Creates a content/project directory if it doesn’t exist
  2. Copies the README.md with proper Hugo front matter
  3. Creates an index file with navigation links
  4. Integrates with the deployment process

The documentation is accessible at:

  • Main documentation: /project/README/
  • Project section: /project/

12.2 Documentation Structure # 

content/
└── project/
    ├── README.md     # Main project documentation
    └── _index.md     # Project section landing page

12.3 Customization #

To customize the documentation integration:

  1. Modify the front matter in content/project/README.md:

    ---
title: 'Project Documentation'
description: "Documentation for Jeff's Link - Personal Knowledge Base"
date: 2024-03-07
draft: false
weight: 1
---

  2. Update the navigation links in content/project/_index.md

  3. Modify the deployment script (scripts/deploy.sh) to change the integration behavior
